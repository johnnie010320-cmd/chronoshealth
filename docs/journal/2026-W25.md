### 요청 1 — 아이폰 세션 유지 (ADR 선행)
- **현상(측정)**: PC는 로그인 유지, 아이폰 Safari는 며칠 만에 로그아웃.
- **원인(가설)**: 세션이 localStorage 저장(`session.ts`) + 게이트웨이 크로스 오리진 → iOS Safari ITP의 스크립트-저장소 7일 캡에 걸림. 서버 토큰은 30일 유효(`tokens.ts`).
- **ADR 0014 (Proposed)** 작성: 세션을 httpOnly·Secure·SameSite=Lax 쿠키로 전환. 전제 = 게이트웨이를 웹과 동일 site(`api-chronoshealth.ever-day.com`)로 이전.
- **착수 대기**: ① 아이폰 7일+ 재현으로 가설 확정 ② 게이트웨이 도메인 이전 승인.

### 요청 1 — 중대 발견으로 착수 중단 (분석 불일치 → 보고)
- ADR 0014의 승인된 접근(same-site 서브도메인 `api-chronoshealth.ever-day.com`)이 **직접 provision 불가**.
- 이유: `ever-day.com` 네임서버는 **Gabia(ns1.gabia.co.kr)** — 존이 Cloudflare에 없음(ADR 0006). CF Worker 커스텀 도메인/라우트는 존이 CF에 있어야 함.
- **수정 단일안(DNS 변경 없음)**: 게이트웨이를 `chronoshealth.ever-day.com/api/*` 경로로 **same-origin**, CF **Pages Function 프록시**로 노출 → 쿠키가 first-party가 되어 ITP 회피. 기존 Pages 프로젝트에 추가, Gabia/존 이전 불필요.
- ADR 0014 슬라이스(서브도메인 → same-origin 프록시) 개정 필요. 게이트웨이 코드는 작성/배포하지 않음(파괴적 변경 회피). 승인 대기.

### 요청 1 — 구현·배포 완료 (2026-06-17, ADR 0014 Accepted)
- 승인된 same-origin Pages Function 프록시 방식으로 구현.
- **게이트웨이**: `auth/cookies.ts`(chronos_session httpOnly + chronos_uid 마커), signup/login/set-password 쿠키 발급, authMiddleware 쿠키 우선·Bearer 폴백, `POST /auth/logout`(revoke+삭제), CORS credentials. 테스트 4건 추가 → **196 전체 통과**.
- **웹**: `functions/api/[[path]].js` same-origin 프록시, api-client 상대경로(`GATEWAY_URL=''`)+submitLogout, session.ts `chronos_uid` 마커로 ITP 대응 세션 복원, 로그아웃 서버 호출.
- 검증: 게이트웨이 196 테스트, 웹 tsc 0 / build 0. 커밋 `9b635db`.
- **배포**: 게이트웨이 Worker(Version `f4017e35`) → 웹 CF Pages(Functions bundle 포함) 순. 무중단(역호환).
- 라이브 스모크: health 200 / 미인증 401 / 프록시 콘텐츠 200 / **logout Set-Cookie 2종 패스스루 + Domain 미지정(host-only=first-party) 확인**.
- **남은 1건**: 아이폰 Safari 7일+ 경과 세션 유지 — 죠니 단말 실측(물리적 시간 필요, 지니 수행 불가).
