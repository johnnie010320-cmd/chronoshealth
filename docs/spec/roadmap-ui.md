- **Related**:

  - `docs/spec/risk-survey/04-report-ui.md` (결과 카드 — R2 CTA 부착 지점)

> 본 슬라이스의 목적: 23문항 결과 카드로 끝나던 사용자 여정에 **"앞으로 무엇이 오는가"를 보여주고, 베타 대기자 명단을 확보**한다. 디지털 트윈 아바타("크로노스") UI 본격 구현은 M7 시뮬레이션 슬라이스 도착 시점에 별도 명세로 분기.

---

## 1. 목표 (Goal)

1. **사용자 — 미래 가치 인지**: 결과 카드를 본 사용자가 "이건 시작에 불과하다"를 즉시 인지 (D-day 시뮬레이션 / 5종 나이 / 랭킹 / 코인 보상이 곧 온다)
2. **사업 — 베타 대기자 확보**: 이메일 수집을 통해 P1~P2 출시 시 즉시 도달 가능한 100~500명 대기자 풀
3. **투자/감사 외부 — 로드맵 가시화**: 외부 이해관계자(투자자·CertiK·LBank·언론)가 단일 URL에서 제품 전체 그림 확인 가능

## 2. 비목표 (Non-goal)

본 슬라이스에서 **의도적 제외**:
- ❌ "내 크로노스" 아바타 시각화 (정적 placeholder만, 본격은 M7 슬라이스에서)
- ❌ D-day 시뮬레이션 What-if (M7 P1 후반)
- ❌ 랭킹 실측 표시 (M6, 베타 1,000명+ 필요)
- ❌ CHRO 토큰 잔액·UI (M10 P4 이후)
- ❌ 베타 등록자 보상 (현 단계 약속 금지 — 증권성·특금법 리스크)
- ❌ 이메일 매직 링크 검증 (R3 1차 미포함, 후속 슬라이스)
- ❌ 베타 등록자 수 공개 / FOMO 카운터 (현 단계 미사용)

## 3. 사용자 흐름 (User Flow)

```

  / (랜딩)  ←─── (헤더에 "로드맵" 링크 추가)
   ↓

```

### Slice R3: `/beta-signup` 등록 (FE + BE)

- **추정**: 2일 (FE 1일 + BE 1일 + 검증 별도)
- **의존**: **ADR 0011 선행 필요** (베타 등록 데이터 모델 — 죠니 승인)
- **FE 파일**:
  - `apps/web/src/app/beta-signup/page.tsx`
  - `apps/web/src/components/beta/SignupForm.tsx`
  - `apps/web/src/components/beta/ConsentChecklist.tsx`
- **FE 필드**:
  - 이메일 (필수, RFC 5322 검증)
  - 국가 (필수, 4언어 드롭다운 — 한·미·일·기타)
  - 연령대 (필수, "19세 미만" 선택 시 즉시 차단 표시)
  - 관심 모듈 (선택 다중, M1~M11 중)
  - 동의 체크박스 3종 (필수):
    - PII 수집 및 처리 동의 (ADR 0003 표기)
    - 의료 진단·처방·치료 아님 확인
    - 향후 보상은 약속이 아니라 검토 중임 확인 (특금법·증권성 가드)
- **BE 파일**:
  - `services/gateway/src/beta/signup.ts` (Hono route)
  - `services/gateway/src/beta/signup.test.ts` (Vitest)
  - `services/gateway/migrations/0011_beta_signups.sql` (D1)
- **BE 엔드포인트**: `POST /api/v1/beta-signup`
  - 요청: `{ email, country, ageGroup, interestedModules[], consents[3] }`
  - 응답: `201 { id, registeredAt }` / `400` / `409 (이미 등록)` / `403 (19세 미만)`
- **데이터 모델 (ADR 0011 초안)**:
  - **`identity` 도메인**(평문 PII, 격리 배포 — ADR 0003): `beta_signup_identity (id, email_plain, ip_hash, user_agent, created_at)`
  - **`analysis` 도메인**(가명화): `beta_signups (id, pseudonym_id, country, age_group, interested_modules, consented_at)`
  - `pseudonym_id = HMAC-SHA256(email + server_salt)` — 동일 이메일 재등록 차단용 (식별값 자체는 평문 미보관)
- **단위 테스트 (≥ 10건)**:
  - 201 신규 등록 / 400 이메일 형식 / 400 필수 동의 누락 / 403 미성년 / 409 중복 / 200 health / CORS preflight / DB 격리 검증 / 금지 표현 회귀 / Rate limit (IP당 1시간 5회)
- **라이브 검증 (3관점)**:
  - Positive: 4언어 각각 신규 등록 (총 4건)
  - Negative: 미성년 차단 / 동의 누락 / 중복
  - 회귀: 기존 23문항 → 결과 카드 골든 패스 통과
- **시빌 가드 (P0 수준)**: IP 해시 + Rate limit. ZK 시빌은 P4 이후.

## 5. ADR 0011 (선행 필수)

**제목**: 베타 등록 데이터 모델 — identity-vault 격리

- **결정**: 이메일 평문은 `identity` 도메인 D1에만, `analysis` 도메인에는 HMAC pseudonym만 저장
- **이유**: ADR 0003 (PII 격리)의 베타 슬라이스 적용. 향후 ML 학습 데이터에서 이메일 평문 분리.
- **결과**: R3 BE 구현 가능. 의료 자문/감사관이 "분리 보관" 검증 가능.
- **거부 대안**:
  - 단일 테이블 평문 보관 → ADR 0003 위반
  - 메일 서비스 위탁(Mailchimp 등) → 베타 단계 외부 위탁은 동의 범위 초과

**Status**: 죠니 승인 시 ADR 0011 작성 → Accepted 후 R3 착수

## 6. 회귀 가드 (기존 기능 100% 정상 유지 전제)

본 슬라이스가 영향 줄 수 있는 기존 기능:

| 영역 | 확인 방법 | 통과 기준 |
|------|-----------|-----------|
| 23문항 폼 | Playwright E2E + 수동 4언어 | 기존 동작 그대로 |
| `/api/v1/risk-estimate` | 16/16 단위 테스트 + 라이브 smoke | 200/401/403/400/429 모두 정확 |
| Slice 05 D1 저장 (consentToStore) | 단위 테스트 통과 | 동일 |
| 다크모드 / i18n 4언어 | 시각 회귀 검증 | 회귀 0건 |
| identity 도메인 (가입 ADR 0010) | 회귀 단위 테스트 | 격리 유지 |

**원칙**: 기존 컴포넌트·API 수정 최소화. 신규 컴포넌트·라우트·테이블로 격리. (회귀 방지 메모리 규범)
