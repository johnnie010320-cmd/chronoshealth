# /slice <기능>

가장 얇은 수직 슬라이스 1개를 1~3일 단위로 추출. 1슬라이스 = "한 사용자 흐름이 인프라부터 UI까지 동작".

## 사전 조건

`docs/spec/<기능>.md` 가 **Accepted** 상태여야 함.

## 출력

```markdown
# Slice — <기능> / <슬라이스명>

- **Spec**: docs/spec/<기능>.md
- **Estimate**: 1~3일
- **Linear**: CHR-XXX (생성 후 채움)

## 완료 정의 (Definition of Done)
- [ ] 사용자가 X 행동을 시작
- [ ] 시스템이 Y 흐름을 거쳐
- [ ] 사용자에게 Z 결과 표시
- [ ] 단위 테스트 통과
- [ ] 통합 테스트 1개 통과
- [ ] 컴플라이언스 / 솔리디티 리뷰 (해당 시)

## 영향 영역
- `apps/web/...` — 무엇을
- `services/gateway/...` — 무엇을
- `services/<...>/...` — 무엇을
- `ml/...` — 무엇을 (해당 시)
- `contracts/...` — 무엇을 (해당 시)
- DB 마이그레이션 — Yes/No

## 의존성
- 선행 슬라이스 (있다면)
- 외부 API / SDK
- 인프라 변경 필요 여부

## 검증 시나리오
- Positive: ...
- Negative: ...
- 회귀: 기존 기능 X 영향 없음 확인
```

## 금지

- 하나의 슬라이스에 두 기능 섞기
- "백엔드만 / 프론트엔드만" 슬라이스 — 수직이 아님
- 1주일 초과 슬라이스 — 더 쪼개기
