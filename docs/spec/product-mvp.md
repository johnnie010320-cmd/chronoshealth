## 1. 제품 한 줄 정의

> **"내 디지털 아바타 ‘크로노스’가 죽지 않도록 키우는, AI 노화·질병 예측 + Health-to-Earn 생존 게임."**

- 핵심 메타포: **디지털 트윈 = 크로노스 아바타**. 사용자의 행동/측정값이 아바타의 **활력 점수(Vitality Score)** 와 **예측 잔여 연수(신뢰구간 포함)** 를 늘리거나 줄인다. (카운트다운/D-XXX UI 절대 금지 — `apps/web/CLAUDE.md` 규칙 5)
- 데이터 주권: 모든 개인 데이터는 **클라이언트 측 암호화 후 분산 저장**(IPFS/Filecoin 등). 크로노스 서버는 평문 보관 금지(절차서 9장).
- 보상: 데이터 제공 + 건강 행동 + 커뮤니티 기여에 **CHRO 토큰**(Polygon PoS) 머클 드롭.
- 출구: **만성질환자가 자신과 같은 환자를 찾아 묻는 곳** — 의사가 답하지 않는 영역의 대체 커뮤니티.

## 2. 핵심 사용자 가치 (Why this exists)

| 가치 | 사용자 체감 | 차별점 |
|------|-------------|--------|
| **수치화된 미래 자아** | "나는 지금 같은 또래 5천만명 중 상위 5%" | 단순 헬스 리포트 X, **랭킹·게임화** |

## 4. 모듈 상세 명세

### M1. 인증·온보딩

**목표**: 사용자가 30초 안에 가입하고 첫 결과 카드까지 도달.

| 항목 | P0/P1 (현재) | P2 | P3+ |
|------|-------------|----|-----|
| 가입 정보 | 이름·전화·이메일 (ADR 0010) | + 생년월일·성별·국가·도시 | + KYC 1~3등급 |
| 로그인 | 이메일만 | Google / Facebook / Kakao / Naver OAuth | + 패스키 / WebAuthn |
| 미성년자 차단 | 만 19세 미만 가입 차단 (절차서 9.3) | 동일 | 동일 |
| 세션 | session-token (서버 발급) | 동일 | + DID 기반 인증 옵션 (M_DID) |

**규제 가드**: PII는 `services/identity/` 외부 유출 금지(ADR 0003). 분석 DB에는 `user_pseudonym_id`만.

### M2. 데이터 입력 1차 (자가 보고 + 웨어러블)

**P1 — 자가 보고 (현재)**:
- 23문항 설문 (`docs/spec/risk-survey/01-survey-form.md`)
- 항목: 인구학(연령·성별·키·체중) + 생활(흡연·음주·운동·수면) + 측정값(혈압·혈당·콜레스테롤 알면 입력)

**P2 — 웨어러블 연동 (확장)**:
- **iOS HealthKit**: 심전도(ECG), 혈압(Apple Watch 시리즈 9+), 혈당(연속 측정기 연동), 활동량(걸음수·운동량), 수면 패턴, 심박변이도(HRV)
- **Samsung Health**: 갤럭시워치 동일 항목
- **Oura Ring / Whoop / Garmin**: 표준 OAuth + REST
- **데이터 빈도**: 실시간 푸시(이벤트형) + 일 1회 배치 (집계형)
- **저장**: 원본은 IPFS 암호화 청크, 집계 통계만 `services/ml-serving` D1/Timescale

**비목표 (절차서 9.2)**:
- 인바디·체성분계 등 비표준 디바이스는 P3 이후
- 의료기기 등급 측정(부정맥 진단 등)은 절대 금지 표현 가드 적용

### M3. 예측 엔진 — 5종 나이 + 질병 위험

**산출물 (사용자 노출)**:
1. **생명 나이** (Life Age) — 통계학적 기대 생존 곡선
2. **젊음 나이** (Vitality Age) — 전반적 활력 종합 점수
3. **피부 나이** (Skin Age) — 카메라 분석 (P3+) 또는 자가 보고 기반
4. **혈관 나이** (Vascular Age) — Framingham + ABI/혈압 기반
5. **관절 나이** (Joint Age) — 활동량 + 자가 보고 통증 + (P3+) IMU 보행 분석

**5종 질병 5년 위험**: 심뇌혈관·당뇨·만성신장병·치매·암 (risk-survey §6 결정)

**페이즈별 정확도 목표**:
- P1: 결정형 (Framingham 정통 + 휴리스틱 4종), 임상 검증 미실시 — 베타 100명 자기평가 만족도 ≥ 70%
- P2: 한국 코호트 보완 (KoGES 등 공개 데이터)
- **P3**: PhenoAge / DeepSurv / Transformer 앙상블, **AUROC ≥ 0.78** (절차서 5.4 P3 종료 조건)
- P4: ZK 증명 기반 추론 (입력 비공개로도 점수 산출 가능)

**금지 표현 (UI/응답 전체)**: "진단" / "처방" / "치료" / "사망일" / "여명" / "죽음" / **"카운트다운"** / **"D-XXX"** / **"예상 소멸일"** / **"사망예상일"** → 허용: "예측 리포트" / "위험 추정" / "건강 위험 추정" / "활력 점수" / "예측 잔여 연수(CI 포함)"

---

### M4. 데이터 입력 2차 (가족력·검진·만성질환)

**P2 영역**:
- 가족력: 부모/조부모 4대 질병 (심혈관·당뇨·암·치매) 발병 여부 + 발병 연령
- 과거 병력: 수술 기록(연·부위), 만성 질환(고혈압·당뇨·갑상선 등), 복용 약물
- 건강검진 결과: 국가검진(국민건강보험공단 PDF 업로드) + 일반/특수 검진 수치 (혈액·소변·심전도 등)
- 환경: 거주 지역(미세먼지·기후), 직업 노출(분진·방사선·교대근무)

**P3 영역**:
- **FHIR R4 자동 연동**: My HealthWay (보건복지부), Apple Health Records (US), 21세기 큐어스법 API

---

### M5. 데이터 입력 3차 (유전체·텔로미어)

**P3 영역**:
- **DNA 분석**: 마크로젠 / GC지놈 / 클리노믹스(국내), 23andMe / Color(해외) 결과 PDF/CSV 업로드 → 파싱
- **텔로미어 길이**: 외부 검사 결과 입력 또는 (P4+) 자체 키트 발송
- **에피지놈 시계**: P4+ (Horvath / Hannum / PhenoAge DNAm) — 외부 검사사 협업

**검증**: 유전체 데이터는 평문 저장 절대 금지. 클라이언트 측 AES-256 후 IPFS 청크. 추론은 ZK 증명 또는 동형암호 (P4+).

---

### M6. 랭킹·리더보드

**계층**:
- **세계**: 동일 연령·성별 인구 중 백분위 (예: "전세계 45세 남성 5,000만 중 상위 5%")
- **국가**: 동일 (예: "미국 45세 남성 225만 중 115위")
- **도시**: 동일 (예: "LA City 45세 남성 2.5만 중 N위")

**지표**:
- 1차: 예상 잔여 수명 (M3 산출)
- 2차: 생체 나이 vs 역연령 차이 (얼마나 더 젊은가)

**보상 (P4 이후 활성)**:
- 월별 상위 1~100위 → CHRO 차등 지급 (1위 1000 → 100위 10 등)
- **유의미한 순위 상승** (예: 월 ±100계단 이상 상승) → CHRO 보상 (반복 어뷰징 방지: ZK 시빌 가드 적용)

**P1 한계**: 베타 100명만으로는 실제 랭킹 의미 없음 → **모의 정규분포 + 익명 동의자 데이터**로 시작. 의미 있는 분포는 P2 후반(웨어러블 연동자 1,000명+) 부터.

### M7. 개인 맞춤 시뮬레이션 (예측 잔여 연수 What-if)

**P1 후반 — 결정형 What-if**:
- 입력: 현재 23문항 응답
- 변경 가능 항목: 운동 빈도/강도, 수면 시간, 음주 횟수, 흡연 여부, 체중 ±N kg
- 출력: 변경 시 5종 나이·질병 위험·**예측 잔여 연수(95% CI 포함)** 변화량 (예: "운동 주3회 + 수면 7시간 + 음주 주1회 → 예측 잔여 연수 +1.8년 (CI ±0.6)")
- **UI 형식 제약**: 카운트다운/D-XXX 절대 금지. 점수 + 구간 형식만 허용 (`apps/web/CLAUDE.md` 규칙 5).

**P2 — 웨어러블 적응형**:
- 실측 활동량 추세를 기반으로 "30분 추가 시 효과" 계산
- 심박수·스트레스(HRV) 관리 시뮬레이션

**P3 — ML 개인화**:
- 동일 코호트 비교 기반 권장 행동 ("당신과 비슷한 45세 남성 중 예측 잔여 연수 +5년(CI 포함) 달성한 100명의 공통점")
- 우선순위 제시 (지금 가장 효과 큰 1가지)

**의료 가드**: 제안은 **약물·치료·진단 금지**. "수면 시간 증가" "운동 빈도 증가" "음주 빈도 감소" 등 라이프스타일 변경만.

### M8. 디지털 트윈 아바타 — "크로노스"

**컨셉**: 사용자의 디지털 트윈은 이름이 **"크로노스"**(Chronos = 시간의 신). 사용자가 키우는 게임 대상.

**핵심 메커닉**:
- 크로노스의 **활력 점수(Vitality Score)** — 행동·측정값 기반 종합 점수 (0~100 스케일)
- **예측 잔여 연수(PYR, Predicted Years Remaining)** — 항상 신뢰구간(예: ±2.4년)과 함께만 표시
- 건강한 행동 → 활력 점수 ↑ + 예측 잔여 연수 상한 ↑
- 해로운 행동 → 활력 점수 ↓ + 예측 잔여 연수 하한 ↓
- 목표: "90세에도 마라톤 / 바디프로필 가능한 크로노스 키우기"

**UI 표현 제약 (절대 규칙, `apps/web/CLAUDE.md` 5)**:
- ❌ 카운트다운 / D-XXX / "X일 남음" / "예상 소멸일" / "사망예상일"
- ✅ 활력 점수 + 점수 추세 그래프 + 예측 잔여 연수 (신뢰구간 표시 필수)

**시각화 단계**:
- P1: 정적 카드 (이름 + 활력 점수 + 예측 잔여 연수 CI + 5종 나이)
- P2: 2D 캐릭터 (성별·연령대 기본 일러스트)
- P3: 3D 아바타 (몸매·피부·자세가 데이터 반영)
- P4 (옵션): **온체인 NFT 발행** — 아바타 소유권 토큰화. CHRO 와는 별도 ERC-721. 거래 가능 여부는 법무 검토 후.

**금지**: 죽음/사망 단정 표현 절대 금지. 항상 "예상" + 신뢰구간.

### M9. 소셜 네트워킹

**P3 후반 시작 — 콘텐츠 플랫폼**:
- **숏폼 영상 업로드** (TikTok 스타일) — 운동 루틴, 식단, 회복 과정, 검진 결과 후기
- 대상 인플루언서: 헬창, 러너, 바디프로필, 시니어 운동 성공자 (이길여 여사형 롤모델)
- 인앱 댓글·질문 활성화

**P4 — 보상 활성**:
- 좋아요·조회수 상위 콘텐츠 → CHRO 보상
- 어뷰징 방지: ZK 시빌 + 디바이스 어테스테이션 (절차서 8.3)

**P3~P4 — 환자 매칭 (핵심 차별점)**:
- 내 크로노스와 **같은 질병·신체조건·연령대** 아바타 검색
- 익명 Q&A: "나와 같은 갑상선저하증 45세 남성들은 어떻게 관리하나?"
- 톤: "전문가 답변" 보다 **"동질 환자의 일상 공유"** 우선
- 고퀄리티 답변/좋아요 多 → CHRO 보상

**규제 가드**: 의료 행위 자문 금지 표시(UI 상단 고정 면책). 약 추천·진단 글 자동 필터링.

### M10. 보상·토크노믹스 (CHRO)

**(절차서 8장 + ADR 0004 정합)**

| 페이즈 | 토큰 상태 | 사용자 체감 |
|--------|----------|-------------|
| P0~P2 | 미발행 | "포인트" 표시 (오프체인 이중 기록, 향후 1:1 전환 보장) |
| **P3** | 오프체인 포인트 → DB 누적 | 누적 잔액 표시 |
| **P4** | **테스트넷 ERC-20 + 머클 드롭 시작** | 사용자가 직접 클레임 (가스비 메타트랜잭션) |
| **P5** | 메인넷 + CertiK 감사 통과 | 동일, 보안 인증 표시 |
| **P6** | **LBank 상장** | **DEX/CEX 매도 가능** |

**보상 발생 트리거** (모든 페이즈 공통, 활성화 시점만 다름):
- 데이터 입력 (1·2·3차 각 단계 완료)
- 웨어러블 연속 7일 동기화 (streak 보너스)
- 랭킹 상위 1~100위 월별
- 유의미한 순위 상승
- 좋아요·조회수 상위 콘텐츠 (M9)
- 고퀄리티 환자 매칭 답변 (M9)

**인앱 사용처 (P6+)**:
- 프리미엄 기능: 응급 SMS 알림(M11) 월 **30 CHRO**
- 보고서 PDF 출력
- 외부 가맹점 입점 (피트니스·건강식품·검진센터 — P6 이후 BD)

**시빌 방지 3중**: (1) 디바이스 어테스테이션 (2) KYC 등급 (3) ZK 술어 — 절차서 8.3

### M11. 시니어 돌봄 (부모님 응급 모니터링)

**P2 영역**:
- 자녀가 부모 계정 페어링 (양방향 동의)
- 웨어러블 임계치: 심박수 이상(>120 / <40), 낙상 감지(Apple Watch), SpO2 급락, 무활동 N시간
- 트리거 → 자녀 등록 번호로 SMS (P2: 무료 / P6: **30 CHRO/월 프리미엄**)

**P3 영역**:
- 단순 임계치 외 **ML 이상치 탐지** (개인 기준선 대비 편차)
- 패턴 알림: "지난주 대비 활동량 40% 감소" 등

**비즈니스 동인**: 자녀가 부모님께 앱 설치 → **양방향 가입자 확장** + 시니어 데이터셋 확보(P3 ML 학습 가치 ↑).

## 5. 신원증명 (DID — W3C)

**M_DID — 별도 모듈**:
- P3 PoC: did:ethr 또는 did:polygonid (Polygon ID)
- P4 메인: 분산원장 등록, KYC 등급과 결합
- 사용처:
  - 다른 dApp/서비스에서 "이 사용자는 본인 확인된 크로노스 사용자" 증명
  - 데이터 제공 동의의 비가역 기록 (감사 추적)
  - 영지식 술어: "나는 40~50세 남성이다" — 정확한 ID 노출 없이

**비목표**: P0~P2 에서는 DID 미사용. 단순 session-token (ADR 0010).

## 6. 지역별 GTM 차별화

| 지역 | 진입 톤 | 핵심 가치 | 마케팅 채널 |
|------|---------|-----------|-------------|
| **한국** | **밈코인 + 게임화** | 의료 접근성 세계 1위 → "재미·랭킹·코인"으로 끌어들임 | 헬창/러너 인플루언서, 카카오톡 공유, 디시·블라인드 |
| **미국** | **무료 의료 정보 + 건강관리** | 의료 접근성 꼴찌 → "병원 못 가는 사람들의 건강 코치" | 레딧 r/longevity / r/biohackers, TikTok 헬스 |
| **일본** | **시니어 케어 + 정밀** | 고령화 1위 → "부모님께 깔아드리세요" | LINE 광고, 시니어 매체 |

**우선순위 결정**: 한국 1차 출시(베타 100명) → 일본 P3 확장 → 미국 P4 확장.
